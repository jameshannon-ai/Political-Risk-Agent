# Output Quality Standard

## Decision First

The brief should state the recommended stance before moving into supporting detail.

## Evidence Before Assertion

Judgements should be tied to source evidence, source weight or a named control trigger.

## Numbers Where Available

Use extracted percentages, dates, vessel counts, premium figures, cargo volumes, licence dates, penalties or payment values wherever sources support them.

## Source Hierarchy Visible

Official, regulator, company, market-data, specialist and reputable news sources should be visibly distinguished.

## Confidence Never Overstated

Curated fallback evidence, missing requirements, stale sources or fast-moving conditions should cap confidence.

## Client-Specific Actions

Recommendations must match the business user: underwriting controls for insurers, transaction controls for lenders, routing controls for operators.

## Every Score Justified

Likelihood, impact, immediacy and confidence should each show evidence basis, supporting sources, reason the score is not higher or lower, and a review trigger.

## Every Caveat Actionable

Caveats should explain what to refresh, verify or escalate.

## Reusable But Not Generic

The reusable engine should produce domain-specific language, source requirements and decision outputs for each case.
